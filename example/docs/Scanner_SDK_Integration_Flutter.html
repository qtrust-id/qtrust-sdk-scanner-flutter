1. Ringkasan

QTrust Scanner Flutter SDK = on-device QR & Barcode scanner untuk Flutter (iOS & Android). Engine scan berjalan di dalam WebView yang memuat UI scanner — SDK membundel UI web secara lokal dan menyajikannya lewat localhost secure origin. Seluruh decode dilakukan on-device (zxing-wasm); tidak ada koneksi server yang dibutuhkan.

Pemakaian inti = satu widget: QtrustScannerView. Pasang di widget tree, beri ScannerConfig, dan terima ScanResult lewat callback onResult. Widget memiliki lifecycle kamera sendiri — kamera dilepas otomatis saat widget dispose.

FiturKeterangan
Tipe scanQR Code (2D), Barcode (1D)
SurfaceQtrustScannerView — Flutter widget berisi WebView, mengisi container
BackendOn-device decoder (zxing-wasm) — tidak ada server
API gaya callbackonResult, onError, onReady, onClose
Outputdata (String), format, boundingBox
DistribusiPaket public pub.dev, tanpa build step

3. Instalasi (pub.dev)

Paket dipublish public ke pub.dev. Tidak perlu token atau registry khusus — tambahkan seperti dependency biasa.

3.1 Tambah paket

flutter pub add qtrust_scanner

atau secara manual di pubspec.yaml:

dependencies:
  qtrust_scanner: ^1.0.3

lalu:

flutter pub get
Isi paket Paket sudah membundel UI scanner web (assets/scanner/) sebagai package asset — tidak ada langkah build atau download tambahan. Cukup pub get.

3.2 License

Dirilis di bawah Apache-2.0 — bebas dipakai & dimodifikasi dengan atribusi & klausa paten standar.

4. Setup Platform (iOS & Android)

Scanner butuh kamera. Karena UI berjalan di WebView via getUserMedia, host wajib mendeklarasikan izin kamera OS, dan di Android juga meminta grant runtime sebelum widget di-mount.

4.1 iOS — ios/Runner/Info.plist

Tambahkan deskripsi penggunaan kamera (wajib, jika tidak app crash saat akses kamera):

<key>NSCameraUsageDescription</key>
<string>Aplikasi memerlukan akses kamera untuk memindai QR & barcode.</string>

4.2 Android — android/app/src/main/AndroidManifest.xml

<uses-permission android:name="android.permission.CAMERA" />

4.3 Android — grant runtime sebelum mount

Android 6+ butuh grant runtime. SDK tidak menarik dependency permission agar tetap ringan — minta izin di host (mis. dengan paket permission_handler) sebelum menampilkan QtrustScannerView: 

import 'package:permission_handler/permission_handler.dart';

Future<bool> ensureCamera() async {
  final status = await Permission.camera.request();
  return status.isGranted;
}
Urutan benar Pastikan izin kamera sudah granted sebelum QtrustScannerView mount. Tanpa izin OS, kamera tampil hitam dan onError melaporkan kegagalan.

5.2 Tampilkan scanner

QtrustScannerView mengisi penuh ruang yang diberikan. Letakkan di body Scaffold (atau halaman penuh). Beri config, type, dan callback hasil: 

Scaffold(
  backgroundColor: Colors.black,
  body: QtrustScannerView(
    config: ScannerConfig(
      vendorConfig: VendorConfig(skipTutorial: true),
    ),
    type: ScanType.qr,
    onResult: (result) {
      debugPrint('${result.data} (${result.format})');
    },
    onError: (error) => debugPrint(error.message),
  ),
);
Beri ukuran pada container Seperti WebView lain, widget butuh ukuran. Pakai di halaman penuh atau bungkus dengan SizedBox/Expanded agar tidak 0-tinggi.

7. Tipe Scan (ScanType)

Parameter type menentukan mode kerja engine — bukan filter hasil. Satu tipe per sesi scanner.

8. Callback & Event

CallbackArgumenKapan dipanggil
onResultScanResultTiap kode berhasil di-decode WAJIB
onErrorScannerErrorError (izin, gagal muat halaman/aset, timeout)
onReady—Scanner siap (overlay loading bisa dibuka)
onClose—Scanner ditutup dari UI web internal
onResult fire berkali-kali Dalam mode SDK, engine men-decode tiap frame dan memanggil onResult berulang selama kode terlihat. Bila ingin "scan sekali lalu kembali", latch hasil pertama dan abaikan sisanya (lihat Bagian 11) — pola sama dengan sampel iOS/Android.

9. Membaca Hasil (ScanResult)

class ScanResult {
  final String data;          // isi QR / barcode
  final String format;        // contoh: "QR_CODE", "EAN_13"
  final BoundingBox boundingBox;
}

class BoundingBox {
  final int x;
  final int y;
  final int width;
  final int height;
}

Contoh penggunaan:

void handle(ScanResult result) {
  final text = result.data;
  final format = result.format;
  final box = result.boundingBox;
  debugPrint('@(${box.x}, ${box.y}) ${box.width}×${box.height} — $format: $text');
}
boundingBox selalu non-null Berbeda dengan SDK web, boundingBox di Flutter tidak pernah null — bila engine tak mengirim lokasi, field jatuh ke 0 (lebar/tinggi 0). data tetap valid.

10. Lifecycle & Cleanup

QtrustScannerView memiliki kamera sendiri. Saat widget dispose (mis. pop halaman), SDK otomatis tear down WebView dan melepas stream kamera — host tidak perlu menghentikannya manual.

PeristiwaEfek
Widget mountLoad halaman scanner bundel, inisialisasi on-device decoder, buka kamera
onReadyKamera siap; overlay loading SDK menghilang
Widget disposeTear down bridge, about:blank, lepas kamera
Best practice navigasi Tampilkan scanner sebagai halaman (route) tersendiri. Saat hasil pertama diterima, pop halaman scanner — widget dispose melepas kamera. Jangan menahan instance scanner hidup di belakang layar.

11. Contoh Lengkap

Halaman scanner penuh dengan pola latch hasil pertama: engine memanggil onResult berulang, jadi kita kunci panggilan pertama dan pop tepat sekali — menghindari over-pop. 

import 'package:flutter/material.dart';
import 'package:qtrust_scanner/qtrust_scanner.dart';

class ScanPage extends StatefulWidget {
  const ScanPage({super.key, required this.config, required this.type});

  final ScannerConfig config;
  final ScanType type;

  @override
  State<ScanPage> createState() => _ScanPageState();
}

class _ScanPageState extends State<ScanPage> {
  bool _handled = false;

  void _finish(ScanResult? result) {
    if (_handled) return;
    _handled = true;
    Navigator.of(context).pop(result);
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: Colors.black,
      body: QtrustScannerView(
        config: widget.config,
        type: widget.type,
        onResult: _finish,                 // latch hasil pertama, lalu pop
        onError: (e) => ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text(e.message)),
        ),
        onClose: () => _finish(null),      // tombol close internal
      ),
    );
  }
}

11.1 Membuka scanner & menerima hasil

Future<void> openScanner(BuildContext context) async {
  final result = await Navigator.of(context).push<ScanResult>(
    MaterialPageRoute(
      builder: (_) => ScanPage(
        config: const ScannerConfig(
          vendorConfig: VendorConfig(skipTutorial: true),
        ),
        type: ScanType.qr,
      ),
    ),
  );

  if (result != null) {
    debugPrint('Hasil: ${result.data} (${result.format})');
  }
}

12.1 Menangani error

onError: (ScannerError error) {
  switch (error.kind) {
    case ScannerErrorKind.permissionDenied:
      // arahkan user ke Settings untuk mengizinkan kamera
      break;
    case ScannerErrorKind.connectionFailed:
      // halaman/aset scanner gagal dimuat — tampilkan retry
      break;
    case ScannerErrorKind.timeout:
      // tidak ada hasil dalam batas waktu
      break;
  }
}

13. Keamanan

  • Izin minimal. Deklarasikan hanya NSCameraUsageDescription / CAMERA — tidak ada izin lain yang dibutuhkan SDK.
  • Origin lokal. UI scanner disajikan dari localhost (secure context) di dalam WebView; bukan dari file://. Tidak ada data keluar dari perangkat.
  • On-device sepenuhnya. Tidak ada koneksi server. Hasil decode tidak meninggalkan perangkat kecuali host app yang mengirimkannya.

14. Troubleshooting

GejalaPenyebabSolusi
Kamera hitam / tidak menyala Izin kamera OS belum di-grant iOS: pastikan NSCameraUsageDescription ada. Android: tambah CAMERA + minta grant runtime sebelum mount
Start lambat / lama loading Aset bundel WebView belum selesai dimuat Pastikan flutter pub get berjalan dan aset paket ter-include; coba hot restart
Hasil terus berulang / back "macet" Engine men-decode tiap frame, onResult fire berkali-kali Latch hasil pertama (bool _handled) lalu pop sekali — lihat Bagian 11
Layar scanner kosong / 0 tinggi Container tanpa ukuran Pakai di halaman penuh atau bungkus Expanded/SizedBox dengan ukuran eksplisit
Re-scan setelah back: layar hitam / lambat Instance lama menahan kamera Tampilkan scanner sebagai route; pop saat selesai agar widget dispose melepas kamera
flutter pub get gagal resolve Constraint Flutter/Dart di bawah minimum Pakai Flutter ≥ 3.22, Dart ≥ 3.4
iOS build error soal kamera saat runtime Info.plist tanpa NSCameraUsageDescription Tambah key tersebut (Bagian 4.1) — iOS meng-crash akses kamera tanpa deskripsi
